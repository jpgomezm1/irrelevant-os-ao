---
name: Cotizaciones siempre con skills especificos
description: Las cotizaciones deben incluir los skills reales definidos para el cliente, nunca ejemplos genericos
type: feedback
---

En la pagina "Que incluye" de las cotizaciones, siempre mostrar los skills especificos que se diseñaron para el cliente, no ejemplos genericos.

**Why:** Sara pidio explicitamente que todas las cotizaciones incluyan los X skills acordados. La cotizacion de MAAS 3 tenia ejemplos genericos y tuvo que corregirse manualmente.

**How to apply:** Al generar cualquier cotizacion con /cotizacion, leer primero la propuesta de skills del cliente (Clientes/[empresa]/diseno/propuesta-skills.md) y usar esos skills con formato Antes/Despues en la seccion de procesos inteligentes. Nunca poner "Estos son ejemplos reales" ni "se definen durante el diagnostico" — ya estan definidos.
