1. EL PROBLEMA

Diagnóstico del proceso actual de importaciones en COMEX

Resumen Ejecutivo

El equipo de importaciones de Campuzano Group procesa actualmente las Declaraciones de Importación (Formulario 500 DIAN) mediante llenado manual casilla por casilla. Este proceso operativo consume recursos significativos, limita la capacidad de procesamiento y genera riesgo de errores que pueden resultar en retenciones costosas en puerto.

Situación Actual Diagnosticada

Proceso Manual Ineficiente

  • 91 casillas del Formulario 500 se llenan manualmente una por una en el sistema DIAN
  • 5-6 documentos fuente diferentes de los cuales se extrae información:
    • Factura comercial
    • Guía aérea / Bill of Lading (BL)
    • Factura de fletes internacionales
    • Certificado de seguro
    • Certificados de origen y sanitarios
    • Manifiesto de carga DIAN
  • Tiempo de procesamiento: 45-60 minutos por declaración
  • Códigos DIAN complejos requieren consulta constante de tablas (ciudades, depósitos, transportadores, formas de pago, tipos de importación)
  • Validaciones manuales: Los términos de negociación requieren interpretación experta (ejemplo: una factura que dice SWORD pero incluye flete debe corregirse a CPT)

Carga Operativa Desbalanceada

3:1

1:10
Persona por exportaciones
6X
Diferencial de eficiencia

Costos del Problema

  • Costo operativo alto: 3 personas dedicadas por cada importación vs 1 persona para 10 exportaciones
  • Capacidad limitada: La velocidad de procesamiento está limitada por digitación manual
  • Riesgo de retenciones: Errores en documentación pueden generar costos de $100-500 USD/día por contenedor retenido en puerto
  • Impacto al cliente: Mayor tiempo de nacionalización = mayores costos de bodegaje y financieros para el importador

Oportunidad No Aprovechada

El sistema DIAN permite carga masiva de declaraciones vía archivo plano Excel, un método significativamente más eficiente que el llenado casilla por casilla. Sin embargo, generar ese archivo Excel de forma manual también es complejo, por lo que actualmente no se aprovecha esta funcionalidad del sistema.

2. LA SOLUCIÓN

Sistema automatizado de generación de Declaraciones de Importación

Objetivo

Implementar un sistema inteligente que extraiga automáticamente la información de los documentos fuente, valide las reglas de negocio DIAN, y genere el archivo plano Excel listo para carga masiva en el sistema de la DIAN, reduciendo el tiempo de procesamiento de 45-60 minutos a 10-15 minutos por declaración.

Flujo del Sistema

Documentos PDF → Motor de IA → Validación automática → Aplicación de reglas DIAN → Archivo Excel listo

Componentes de la Implementación

Componente 1 — Automatización de Declaraciones de Importación (DIM)

5 Procesos Inteligentes:

  • Extractor de documentos: Extrae datos de factura comercial, guía aérea/BL, factura de fletes, certificado de seguro y manifiesto DIAN
  • Validador de términos: Interpreta y valida términos de negociación (SWORD, CPT, FOB, CIF, etc.) según reglas DIAN
  • Asistente de códigos DIAN: Aplica automáticamente códigos de ciudades, depósitos, transportadores, formas de pago y tipos de importación
  • Validador de reglas de negocio: Valida DAV, forma de pago, cálculos de arancel e IVA
  • Generador de archivo plano Excel: Produce el Formulario 500 completo listo para carga masiva DIAN

Resultado: Reducción de 75% en tiempo de procesamiento (de 45-60 min a 10-15 min)

Componente 2 — 2 Skills Adicionales Personalizadas

Por este costo de implementación, construimos 2 procesos inteligentes adicionales para otros flujos operativos de Campuzano Group.

¿Qué es una Skill?
Una skill es un proceso automatizado que resuelve una tarea específica dentro de la operación. Funciona como un asistente especializado que ejecuta un flujo completo de forma autónoma.

Ejemplos disponibles:

  • Skill de seguimiento a cliente (respuesta automática de trazabilidad)
  • Skill de validación pre-importación (identifica requisitos DIAN antes de embarcar)
  • Skill de generación de cotizaciones (calcula aranceles e IVA automáticamente)
  • Skill de alertas operativas (monitoreo de plazos críticos)

Las 2 skills se definen en conjunto con Campuzano durante el Día 1 del proyecto, según las prioridades operativas más allá de la DIM.

Componente 3 — Stack de Herramientas Configurado

Infraestructura instalada y lista para usar:

  • Claude Plan Max: Motor de inteligencia artificial para procesamiento de lenguaje natural y extracción semántica
  • API de OCR: Sistema de procesamiento de PDFs y extracción de datos de documentos escaneados
  • Base de datos de códigos DIAN: Catálogo completo de códigos de ciudades, depósitos, transportadores, formas de pago
  • Generador de Excel: Motor de generación automática de archivos planos compatibles con sistema DIAN

Incluye además:

  • Documentación completa del sistema
  • Manual de uso y validación de outputs
  • Sesión de entrenamiento (2-3 horas) para el equipo
  • Soporte durante la primera semana de producción

Preguntas Frecuentes

¿Qué pasa si necesitamos más skills después de la implementación?

Podemos construir skills adicionales mediante proyectos independientes. Cada skill adicional se cotiza por separado según complejidad.

¿El sistema funciona sin conexión a internet?

No, requiere conexión a internet para consultar las APIs de OCR y Claude. El volumen de datos es bajo (~10-20 MB por declaración).

¿Qué sucede si cambian las reglas DIAN?

Actualizaciones por cambios normativos DIAN están fuera del alcance de esta implementación. Requieren contrato de soporte y mantenimiento por separado.

¿Podemos escalar el sistema a más volumen de operaciones?

Sí, el sistema escala según el volumen. El costo mensual de tecnología aumentará proporcionalmente al número de declaraciones procesadas.
